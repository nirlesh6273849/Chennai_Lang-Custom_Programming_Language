How Chennai Lang Works

What if your programming language had Chennai slang based syntax, supported both {} blocks as in C, Java and indentation-based blocks as in Python, and allowed operators to be defined with custom behavior?

That's why I built a custom programming language featuring lexical analysis, recursive descent parsing, AST generation, bytecode compilation, and a stack-based virtual machine. Designed the compiler pipeline, runtime behavior, and execution architecture from scratch in C++, with extensible support planned for WebAssembly execution.

Compiler Pipeline

📝
Source Code
.ch files written in Chennai Lang syntax
🔤
Lexer (Tokenizer)
Breaks source into tokens: keywords, literals, operators, identifiers
🌳
Parser (Recursive Descent)
Validates grammar, builds structured representation from token stream
🏗️
AST (Abstract Syntax Tree)
Tree representation of program structure: expressions, statements, functions
🔧
Intermediate Repr. (IR)
Platform-independent bytecode instructions for the virtual machine
⚡
Optimizer
Constant folding, dead code elimination, expression simplification
💻
Machine Code (Execution)
Stack-based VM executes bytecode, manages memory, produces output
